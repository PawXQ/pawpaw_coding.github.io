Day 41 - Web Foundation - Introduction to HTML

Internet

Internet 本質是上是連接全球電腦的網路, 主要組成：

  • Server(服務器)：24/7在線提供數據
  • Client(客戶端)：用戶端設備
  • ISP(網絡服務提供商)：提供 internet 接入服務(中華)
  • DNS Server：將網址轉換為IP地址的系統

Data Transfer

  • 通過海底電纜連接各大洲
  • 使用光纖技術
  • 傳輸速度可達400GB/秒
  • 通過IP地址定位設備

上網過程

輸入 url => DNS查詢IP地址 => 服務器返回HTML、CSS、JavaScript文件 => browser 解析並呈現網頁內容

Website

  1. HTML

    • 負責網站結構
    • 定義頁面元素（如按鈕、圖片、文本框）

  2. CSS

    • 控制網站外觀
    • 設定顏色、字體、佈局等視覺效果

  3. JavaScript

    • 實現網站功能
    • 添加互動行為
    • 處理用戶操作

HTML

HTML (Hypertext Markup Language) 是網站的基礎語言

負責定義網頁的內容和結構, 它是唯一可以單獨創建網站的語言.

只有 CSS 或 JavaScript 建不出網頁

組成部分解析

Hypertext(超文本)

  • 可連結到其他文件的文本
  • 通過藍色超連結呈現
  • 實現文件間的相互連接

Markup Language（標記語言）

  • 使用標籤(tags)標記內容
  • 類似編輯標記的概念
  • 定義內容的展示方式

Heading Element

基本語法

1

<h1>Hello World</h1>
  • opening tag：<h1>
  • closing tag：</h1>（注意斜線/）
  • content：放置於開始和結束標籤之間

術語定義

  • Tag(標籤)：<>(尖括號)內的代碼，如<h1>和</h1>
  • Element(元素)：完整結構, 包含opening tag、closing tag和 content

Heading層級

1
2
3
4
5
6

<h1>一級標題</h1>
<h2>二級標題</h2>
<h3>三級標題</h3>
<h4>四級標題</h4>
<h5>五級標題</h5>
<h6>六級標題</h6>

特點：

  • h1最大，h6最小
  • 僅支持h1-h6，無h7以上標籤

使用規範

  1. 每個頁面僅使用一個h1標籤
  2. 不跳過層級（如h1後直接使用h3）
  3. 按層級順序使用（h1 → h2 → h3）

MDN Web Docs HEML

Paragraph Element

基本語法

1

<p>這是一個段落內容</p>
  • opening tag：<p>
  • closing tag：</p>
  • content：放在標籤之間

使用目的

  1. 視覺分隔：自動在段落間添加間距
  2. 無障礙支持：幫助螢幕閱讀器識別段落結構
  3. 避免文本擠在一起

效果對比

Second paragraph content

<p>Second paragraph content</p>
讓我仔細思考這篇文章的重點：

首先,這是關於HTML void elements的教學內容。主要介紹了兩個重要的void elements: hr和br。

關鍵概念:

  1. void elements的定義 - 不允許有內容的HTML元素
  2. void elements的獨特語法 - 自閉合標籤,前斜線位置特殊
  3. hr元素的用途 - 用於分隔內容的水平線
  4. br元素的用途 - 在段落內強制換行,如詩歌和地址

需要特別強調的技術細節:

  • 標籤語法中斜線(/)的正確位置
  • void elements只有單一標籤,沒有開始和結束標籤
  • HTML5中結尾斜線變為可選

最佳實踐建議:

  • br應該只用於段落內部換行
  • 不同段落應使用p元素而非br
  • 建議保留結尾斜線以提高代碼可讀性

讓我用簡潔但完整的方式組織這些信息。

HTML Void Elements

定義

Void elements 是特殊的 HTML 元素, 不會包含任何 content

語法特點

<br /> <!-- 推薦寫法 -->

常用 Void Elements

Horizontal Rule (hr)

  • 用途：創建水平分隔線
  • 示例：
1
2
3

<p>第一段落</p>
<hr />
<p>第二段落</p>

Break (br)

  • 用途：在段落內強制換行
  • 示例：
1
2
3
4
5

<p>
  123 Main Street<br />
  New York, NY 10001<br />
  USA
</p>

最佳實踐

  1. 段落分隔使用 <p> 而非 <br />
  2. <br /> 僅用於段落內必要的換行
  3. 建議保留結尾斜線(/)以提高代碼可讀性
  4. 注意斜線方向：使用正斜線(/)而非反斜線()
